## 📚 Idea/Concepto 

IaaS: Se alquilan máquinas virtuales, discos y redes. El usuario instala y administra el sistema operativo, las aplicaciones y los datos. Está diseñado para administradores de IT que necesitan máximo control.
PaaS: Se alquila todo lo necesario para crear aplicaciones. El proveedor gestiona el sistema operativo, el runtime y el middleware; el desarrollador solo aporta su código y sus datos. Está diseñado para desarrolladores que no quieren gestionar infraestructura.
SaaS: Se alquila la aplicación ya lista para usarse, se accede por internet y no hay que preocuparse ni por las máquinas ni por las actualizaciones. Está diseñado para usuarios finales.
La elección entre modelos es un equilibrio entre control y overhead operativo: a mayor abstracción (SaaS), menor es la carga de gestión para el equipo, pero también se pierde control sobre la infraestructura subyacente.
